# System Architecture

## Core System Idea

The Middle should function as a central knowledge and intelligence operating system.

It receives raw information, organizes it into structured records, links related entities, and turns scattered material into usable knowledge.

## Platform Layers

### 1. Source Layer

Stores raw materials:

- documents
- videos
- webpages
- notes
- images
- evidence
- legal records
- meeting records

### 2. Archive Layer

Preserves files with metadata, source references, ownership, dates, and confidence levels.

### 3. Entity Layer

Extracts core objects:

- person
- organization
- location
- case
- document
- evidence
- event
- concept
- project

### 4. Relationship Layer

Connects entities into a graph.

Examples:

- person -> organization
- person -> case
- case -> document
- document -> evidence
- concept -> concept
- organization -> project

### 5. Timeline Layer

Orders events, documents, and decisions by time.

### 6. Analysis Layer

Compares sources, identifies patterns, detects contradictions, and prepares summaries.

### 7. Decision Support Layer

Presents human-readable options, risks, unknowns, and recommended next steps.
