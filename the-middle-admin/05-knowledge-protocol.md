# Knowledge Protocol

## Principle

Knowledge inside The Middle must be structured, traceable, and reviewable.

## Intake Steps

1. Capture the source.
2. Record metadata.
3. Summarize neutrally.
4. Extract entities.
5. Extract events.
6. Extract claims.
7. Link evidence.
8. Mark confidence.
9. Add open questions.
10. Decide next action.

## Classification

Each item should be tagged by type:

- document
- note
- evidence
- event
- person
- organization
- location
- case
- concept
- decision
- task

## Source Quality

Use these levels:

- primary source
- official source
- direct record
- witness statement
- internal note
- external commentary
- AI-generated draft
- unknown

## Claim Handling

Each important claim should be stored separately from the document summary.

Claim fields:

- claim text
- claimant
- source
- date
- related entities
- supporting evidence
- contradicting evidence
- verification status
- confidence

## Verification Status

- verified
- partially verified
- unverified
- disputed
- false
- unknown

## Naming Convention

Use clear filenames:

`YYYY-MM-DD_type_short-title_source.ext`

If the date is unknown:

`undated_type_short-title_source.ext`

## Review Rule

No high-impact decision should rely on a single unverified source.
