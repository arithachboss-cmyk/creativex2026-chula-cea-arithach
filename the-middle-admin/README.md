# The Middle Admin Seed

Project: The Middle / เดอะมิดเดิ้ล

Core statement:

> องค์กร เดอะมิดเดิ้ล คือ ตัวกลางแห่งทุกสรรพสิ่ง

This folder is the starting operating package for the Admin Agent. It turns the handoff seed into a working structure for knowledge organization, documentation, entity mapping, relationship analysis, and future platform development.

## Purpose

The Middle is a founder-driven knowledge and intelligence initiative. Its role is to connect people, ideas, documents, events, evidence, places, cases, organizations, and systems into a long-term knowledge base.

The AI layer supports the work as a mediator of information. It helps organize, summarize, connect, and reason over material, but it is not the final authority of truth.

## Folder Map

- `00-admin-charter.md` - role, authority, operating rules, and boundaries for the Admin Agent.
- `01-project-brief.md` - concise project identity, mission, vision, and development objective.
- `02-system-architecture.md` - platform modules and system layers.
- `03-data-model.md` - entities, relationships, metadata, and graph structure.
- `04-roadmap.md` - phased development plan.
- `05-knowledge-protocol.md` - how information should be collected, verified, stored, and linked.
- `06-inbox-template.md` - intake template for new documents, cases, events, and concepts.
- `07-decision-log.md` - project decision register.
- `08-glossary.md` - Thai/English concept glossary.
- `09-source-registry.md` - first registry for websites, videos, files, notes, and other sources.
- `10-entity-registry.md` - starting canonical records for organizations, locations, projects, and concepts.
- `11-case-registry.md` - prepared structure for case tracking.
- `12-admin-taskboard.md` - immediate admin work queue.
- `sources/` - incoming source files.
- `archive/` - preserved and normalized records.
- `graph/` - graph exports and machine-readable relationship files.
- `reports/` - generated summaries, briefs, and review documents.

## Immediate Next Step

Use `06-inbox-template.md` for every new source. Once a source is captured, extract entities into the model in `03-data-model.md`, then link them through events, documents, evidence, and concepts.

The first machine-readable graph seed is available at `graph/the-middle-seed-graph.json`.
